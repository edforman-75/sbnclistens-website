## Recommended 7-8 Pages to Fix

Based on user research and project scope, prioritize these pages:

### 1. **Events/Calendar Page** (CRITICAL)
**Fixes:**
- Add search/filter functionality by committee, date, keyword
- Add event category filters
- Show visual indicators for full/open/waitlist status
- Improve layout for easier scanning

**Why:** Most frequent pain point, affects event discovery (primary user goal)

### 2. **Member Dashboard/Profile Page** (CRITICAL)
**Fixes:**
- Show "My Registered Events" prominently
- Make upcoming events easy to find
- Add quick links to most-used functions

**Why:** Users currently can't find their registrations, rely on email

### 3. **Main Navigation (Header/Menu)** (CRITICAL)
**Fixes:**
- Simplify navigation structure
- Reduce number of top-level items
- Improve information architecture
- Add breadcrumbs to reduce backtracking

**Why:** 4 mentions of navigation difficulties, affects all pages

### 4. **Event Detail Pages** (HIGH PRIORITY)
**Fixes:**
- Clarify registration status display (open/full/waitlist)
- Improve clarity of event descriptions
- Show who's registered (if privacy settings allow)
- Make terms and conditions more readable

**Why:** Information quality issues create confusion

### 5. **Committees Page** (MEDIUM PRIORITY)
**Fixes:**
- Rewrite committee descriptions for clarity
- Add visual organization by category
- Make it easier to understand what each committee does
- Add awareness features for specialty committees (e.g., 20-40 somethings)

**Why:** Users lack awareness and understanding of committees

### 6. **Board/About Page** (MEDIUM PRIORITY)
**Fixes:**
- Make board member role descriptions easily accessible
- Improve findability of governance information
- Add clear descriptions of what each board role does

**Why:** Current descriptions are not obvious/accessible

### 7. **Newsletter Archive Page** (QUICK WIN)
**Fixes:**
- Create archive page for past newsletters
- Organize by date
- Make searchable if possible

**Why:** Users want to reference past information, relatively easy to implement

### 8. **Home Page** (FOUNDATIONAL)
**Fixes:**
- Ensure key tasks are easy to start from home
- Improve visual hierarchy
- Add quick access to most common needs (find events, see registrations)
- Reduce clutter

**Why:** Entry point for all users, sets tone for entire experience

## Design Principles for Fixes

Based on user research, prioritize:

### 1. **Utility Over Aesthetics**
- *"I want to do my business and I want to get out"*
- *"Cleaner, simpler, better is my approach"*
- Focus on task completion speed, not visual complexity

### 2. **Clarity Over Completeness**
- Users skip unclear terms and conditions
- Event descriptions need to be transparent and clear
- Don't hide information in nested menus

### 3. **Reduce Clicks and Backtracking**
- Most common complaint: too many steps to accomplish tasks
- Minimize navigation depth
- Remember user context (e.g., position on calendar)

### 4. **Show, Don't Make Users Search**
- Users expect to SEE their registrations, not hunt for them
- Event status should be immediately visible
- Important information should be surfaced, not buried
