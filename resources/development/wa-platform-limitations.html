Wild Apricot Platform Limitations

What even a sophisticated designer cannot change in WA

Overview Widget Gallery Calendar & Photos Strategy Customization Techniques Platform Limitations

1. Purpose of This Document

This document is written specifically for designers who may be accustomed to high-freedom CMS platforms such as Webflow, WordPress, Craft, or custom React front-ends. Wild Apricot is powerful for membership management but structurally limited as a design system.

2. Fixed Page and Layout Structure

All WA themes share the same fixed page model that cannot be overridden:

  • Header region
  • Main menu region
  • Main content region (containing gadgets)
  • Footer region

You cannot replace WA’s templating system or introduce your own layout engine.

3. Gadget-Based Rendering Only

WA outputs all dynamic data through gadgets (widgets). Designers cannot replace or override their HTML templates.

4. System Pages Cannot Be Rebuilt

Certain pages—event pages, registration pages, membership application, profile pages—cannot be rebuilt from scratch.

5. Theme-Level Responsive Behavior

WA themes control:

  • Menu collapse behavior
  • Grid breakpoints
  • Column stacking

Designers cannot adopt alternate responsive frameworks (Bootstrap, Tailwind, custom grids).

6. JavaScript Injection Limits

Custom JS can augment WA pages but cannot modify internal gadget logic or intercept WA AJAX flows.

7. External Embeds & IFrames

Embeds are powerful but introduce:

  • Uptime dependencies
  • Mobile scrolling quirks
  • Touch/keyboard issues
  • SEO invisibility

8. API-Driven Custom UI

Building your own “replacement widgets” using the API is possible, but requires:

  • A backend for secure API calls
  • Managing auth, tokens, errors, failures
  • Owning cross-browser behavior entirely

9. Cross-Browser & Device Compatibility

The more customization added, the more SBNC—not WA—must test and maintain compatibility across browsers and devices.

10. How This Document Should Be Used

This is a boundary document that tells designers what cannot be changed, so they can focus creativity where it will work reliably.
