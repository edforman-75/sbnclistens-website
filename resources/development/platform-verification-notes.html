Platform Verification Notes

Technical reference for identifying whether a website is native Wild Apricot, WordPress, or a hybrid integration. Use these techniques when evaluating other organizations' websites.

How to Verify a Website Platform

When researching other organizations' websites, use these methods to determine what platform they're built on:

Method 1: View Page Source

  1. Right-click on the page and select "View Page Source"
  2. Search for platform-specific indicators (see below)
  3. Look in the <head> section for meta tags and scripts

Method 2: Check URLs

  1. Navigate through the site and observe URL patterns
  2. Wild Apricot: URLs often contain /sys/, /admin/, or /event-
  3. WordPress: URLs may show /wp-content/, /wp-includes/

Method 3: Inspect Developer Tools

  1. Press F12 or right-click → Inspect
  2. Check Network tab for loaded resources
  3. Look for platform-specific JavaScript libraries

Platform Indicators

Wild Apricot Native Site Indicators

  • Meta tags: <meta name="generator" content="Wild Apricot">
  • URL patterns: /sys/, /admin/, /event-, /Sys/
  • Footer text: "Powered by Wild Apricot" (often hidden in small text)
  • HTML classes: wa-widget, wa-event, WaGadgetBlog
  • JavaScript: wildapricot.org/widget/, /Sys/PublicProfiles/

WordPress Site Indicators

  • Meta tags: <meta name="generator" content="WordPress">
  • URL patterns: /wp-content/, /wp-includes/, /wp-json/
  • HTML classes: wp-*, wordpress, post-, page-
  • JavaScript: wp-includes/js/, wp-content/themes/
  • RSS feeds: /feed/, /comments/feed/

Hybrid Integration Indicators

  • Mixed indicators: Both WordPress and Wild Apricot signatures present
  • URL switching: Domain changes between sections (e.g., example.com → example.wildapricot.org)
  • Design inconsistency: Visible style changes when navigating to member areas
  • iframes: Wild Apricot content embedded in WordPress pages

Quick Verification Checklist

Use this checklist when evaluating a new site:

  1. View page source - Look for generator meta tag (usually in first 50 lines)
  2. Check footer - Look for "Powered by" text or small attribution links
  3. Navigate to member area - See if URL or design changes significantly
  4. Inspect events page - Wild Apricot events have distinctive URL patterns
  5. Check mobile responsiveness - Consistent mobile experience suggests single platform

Why This Matters

Platform verification helps the technology committee:

  • Learn from peers: See what other newcomers clubs are using successfully
  • Verify claims: Confirm whether impressive sites are truly native Wild Apricot
  • Assess feasibility: Determine if custom designs require WordPress or work within WA
  • Make decisions: Compare actual platform capabilities vs. vendor claims
  • Set expectations: Understand what's achievable with current platform
