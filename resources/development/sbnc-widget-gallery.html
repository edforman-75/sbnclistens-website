1. How to Read This Page

This gallery lists the main tools we can embed inside Wild Apricot pages. It is organized into:

  • Native Wild Apricot widgets – built into WA, safest for data that lives in WA.
  • 3rd-party widgets – external tools embedded via iframe or script.

For each widget type we note:

  • What it is best at.
  • Key limitations (filtering, styling, mobile behavior).
  • When SBNC should prefer it vs. look at alternatives.

2. Native Wild Apricot Widgets

Native widgets are the safest starting point. They inherit theme styling, respect WA’s permission model, and are maintained by Wild Apricot when the platform changes.

2.1 Event List and Event Calendar

Standard WA event gadgets that show upcoming events in list or calendar views.

  • Strengths:
    • Tightly integrated with WA’s event module and registrations.
    • Respects member vs public visibility rules.
    • Works with system emails and registration flows.
  • Limitations:
    • Filtering can be coarse. Fine-grained views (“only social events”, “only hiking”) may require multiple gadgets or custom views.
    • Visual design is controlled by the theme with only limited tweaks in settings.
    • Month-view calendar layout can feel cramped on small screens.
  • Typical SBNC usage:
    • Primary events page for members.
    • Simple public “upcoming events” listing.
  • Docs: Wild Apricot help center – search for “event calendar gadget” and “event list gadget”.

2.2 Member Directory and Member Profile Widgets

Widgets that expose member lists and profile lookup, respecting privacy settings.

  • Strengths:
    • Fully respects member privacy and contact permissions.
    • Search and filter by standard member fields and selected custom fields.
    • Integrated with login and profile pages.
  • Limitations:
    • Layout options are constrained; advanced card or grid layouts are hard without CSS.
    • Filtering UI is basic compared to a dedicated directory product.
    • No built-in “social graph” features (favorites, follow, etc.).
  • Typical SBNC usage:
    • Member-only directory with simple search and filters.
    • Profile view links from internal communications.
  • Docs: Wild Apricot help center – search for “member directory gadget”.

2.3 Login Box and Authorization Widgets

Login box, “logout” link, “change profile” link, and other authorization gadgets.

  • Strengths:
    • Handle authentication correctly and securely inside WA.
    • React automatically when a user logs in or out.
    • Support redirects to profile pages and member-only content.
  • Limitations:
    • Placement is tied to theme regions; moving login into unusual page areas can be awkward.
    • Visual customization is limited without CSS.
  • Recommendation for SBNC: Always use native login gadgets rather than external authentication tools.

2.4 Content and Media Widgets

Includes content gadgets such as:

  • Content / HTML gadget (rich text, images, embeds).
  • Picture and Slideshow gadgets.
  • File list gadgets (links to documents).
  • Blog and “recent blog posts” gadgets.
  • Strengths:
    • Easy to use for non-technical editors once templates are set up.
    • Slideshow and picture gadgets provide basic galleries without 3rd-party tools.
    • Blog gadgets can power simple news or update sections.
  • Limitations:
    • Photo galleries lack advanced filtering, tagging, or face recognition.
    • Slideshow behavior and layout are relatively rigid.
    • File lists are functional but not visually rich.
  • Docs: Wild Apricot help center – search for “content gadget”, “picture gadget”, “slideshow gadget”, “blog gadget”.

2.5 Donations, Invoices, and Financial Widgets

Donation forms, payment buttons, and invoice/payment history gadgets.

  • Strengths:
    • Tightly integrated with WA’s contacts and financial reports.
    • Respects member records and payment statuses.
    • Less room for errors than custom payment integrations.
  • Limitations:
    • Design flexibility is limited.
    • Advanced fundraising features (campaign thermometers, peer-to-peer pages) require outside tools.
  • SBNC usage: As-needed for donations or specific financial flows; keep inside WA whenever possible.

3. 3rd-Party Widgets and Embeds

3rd-party widgets allow SBNC to go beyond WA’s built-in capabilities. They usually require an iframe or script embed in a Content/HTML gadget and may rely on WA’s API or webhooks for deeper integration.

For each tool, we consider:

  • What it adds beyond native WA widgets.
  • Complexity of setup and maintenance.
  • Impact on reliability, browser behavior, and mobile usability.

3.1 Calendars

  • NewPath CalendarPlus (WA-focused)
    • Pros:
      • Designed specifically for Wild Apricot data.
      • More flexible layouts and filtering than native WA calendar.
      • Can present multiple event types and categories more clearly.
    • Cons:
      • Additional subscription cost.
      • Relies on an external provider; outages affect the calendar view.
      • Styling may not match the theme perfectly without extra work.
  • Tockify Calendar
    • Pros:
      • Modern, visually rich calendar layouts.
      • Good mobile views and “agenda” style lists.
    • Cons:
      • Events must either sync from WA or be maintained separately.
      • Permissions and “member-only” views are more complex.
      • Another external dependency to monitor.

For SBNC, 3rd-party calendars are most attractive if we cannot get acceptable filtering and display behavior from native WA event gadgets, especially for public-facing views.

3.2 Forms and Surveys

  • Jotform (and similar tools)
    • Pros:
      • Very flexible form layouts and logic.
      • Supports conditional questions, file uploads, and workflows.
      • Embeds easily into a WA content gadget.
    • Cons:
      • Data lives outside WA unless synced via export, API, or automation tools.
      • Requires separate account management and access control.
      • Styling may differ from the WA theme if not tuned.

Best for complex surveys or special projects where WA’s built-in forms are too limited, especially when data integration can be handled via Make, Zapier, or manual exports.

3.3 Photo Galleries

  • SmugMug, Flickr, Juicebox, and similar gallery tools
    • Pros:
      • Much stronger support for albums, tagging, and high-quality image display.
      • Some tools support face recognition and member tagging.
      • Better long-term storage and download options for members.
    • Cons:
      • Photos live outside WA; uploads and album management happen in a separate tool.
      • Permissions must be coordinated between the gallery platform and WA.
      • Embedding uses iframes; behavior can vary slightly across browsers and devices.

For SBNC, 3rd-party galleries are strong candidates if we want:

  • Member-downloadable photos.
  • Face recognition or rich tagging of events and people.
  • Long-term archival and backup beyond WA.

3.4 Social and Maps Widgets

  • Elfsight Instagram or similar social feeds
    • Pros:
      • Visually engaging content pulled from existing social channels.
      • Keeps the site looking fresh without manual updates.
    • Cons:
      • Dependent on social platform APIs and rate limits.
      • Can slow page load if not configured carefully.
  • Google Maps or Mapbox embeds
    • Pros:
      • Clear location display for venues and meeting points.
      • Well understood by members.
    • Cons:
      • Privacy and tracking considerations for embedded maps.
      • Additional API keys or usage limits may apply for advanced use.

4. Native vs 3rd-Party: How to Decide

At a high level:

  • Start with native WA widgets whenever:
    • The data already lives in WA (events, members, registrations, donations).
    • We need reliable permissions and login-aware behavior.
    • We want the lowest maintenance burden for future SBNC volunteers.
  • Consider 3rd-party widgets when:
    • We need features WA simply does not have (advanced filtering, tagging, face recognition, rich galleries).
    • We can tolerate another vendor and are willing to maintain API keys or automation workflows.
    • The member experience is meaningfully better and worth the extra complexity.

In customization planning, the Website Committee should:

  • Default to native WA widgets for core membership and event flows.
  • Use 3rd-party widgets selectively for high-impact experiences (e.g., calendar presentation, photo galleries).
  • Document any 3rd-party dependencies, including who owns the account and how data is backed up.
